# Лабораторная работа №4  
## Оконные функции для анализа данных  

**Номер варианта:** 13  

---

# Цель работы

Изучить концепцию оконных функций в SQL и научиться применять их для выполнения сложных аналитических расчетов: ранжирования, вычисления скользящих средних, нарастающих итогов и сравнительного анализа строк без группировки данных.

---

# Задачи

- Разобрать синтаксис оконных функций: `OVER`, `PARTITION BY`, `ORDER BY`, `ROWS` / `RANGE`.
- Научиться использовать функции ранжирования: `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`, `NTILE()`.
- Освоить функции смещения: `LAG()`, `LEAD()`.
- Применить агрегатные функции в качестве оконных для вычисления накопительных итогов и скользящих статистик.

---

# Индивидуальные задания

## Задание 1  

Пронумеровать все дилерские центры по географической долготе (`longitude`) с запада на восток.

Для нумерации используется функция `ROW_NUMBER()`. Так как требуется нумерация от запада к востоку, а долгота увеличивается с запада на восток (западные значения отрицательные, восточные положительные), сортировка выполняется по возрастанию `longitude`.

Скриншот выполнения запроса:

## Задание 2  

Для каждой продажи вывести сумму продажи и сумму следующей продажи в этом же дилерском центре.

Для доступа к следующей строке в рамках одного дилерского центра используется функция ``LEAD()`. Разбиение на окна происходит по `dealership_id`, а внутри окна данные сортируются по дате сделки, чтобы определить, какая продажа является следующей по хронологии.

Скриншот выполнения запроса:
[задание 4.2](https://raw.github.com/MakhoninaAV/sql_files/main/04_02.jpg)

## Задание 3  

Нарастающий итог выплаченных бонусов (предположим 10% от `sales_amount`) для каждого продавца.

Сначала вычисляем размер бонуса как 10% от суммы продажи (`sales_amount * 0.10`). Затем, используя агрегатную функцию `SUM()` в оконном режиме, считаем нарастающий итог (`running total`) бонусов для каждого продавца (`customer_id`). Строки упорядочиваем по дате продажи, чтобы итог накапливался хронологически.
